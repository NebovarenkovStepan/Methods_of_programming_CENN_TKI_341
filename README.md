# Больничный кластер

Проект моделирует распределенную информационную систему медицинского учреждения. В системе есть портал пациента и врача, лабораторный сервис, сервис аптеки, сервис LLM-отчетов и общая PostgreSQL база данных.

## Назначение

Система демонстрирует сквозные процессы больничного кластера:

- создание пациента и ведение электронной медицинской карты;
- запись пациента на прием к врачу;
- назначение лабораторных исследований;
- регистрация образца, выполнение анализа и передача результата;
- генерация текстового медицинского отчета LLM-сервисом;
- создание рецепта, получение его аптекой и выдача лекарства;
- мониторинг оборудования и самодиагностика лабораторных устройств;
- контроль состояния сервисов и складских остатков.

## Архитектура

| Компонент | Технология | Назначение |
| --- | --- | --- |
| `portal-go` | Go | Портал для пациентов и врачей: пациенты, ЭМК, записи на прием, назначения, рецепты |
| `pharmacy-go` | Go | Аптека: получение рецепта, сканирование QR/кодов, выдача лекарства |
| `laboratory-python` | Python/FastAPI | Лаборатория: образцы, анализаторы, рабочие станции, оборудование, мониторинг |
| `llm-python` | Python/FastAPI | Формирование текстовых отчетов по результатам исследований |
| `postgres` | PostgreSQL | Центральная база данных |
| `test` | Python/pytest | Системные end-to-end тесты |

## Основные сервисы

При запуске через Docker Compose сервисы доступны так:

| Сервис | Внутри Docker | На хосте |
| --- | --- | --- |
| Portal | `http://portal:8080` | `http://localhost:8080` |
| Pharmacy | `http://pharmacy:8081` | `http://localhost:8081` |
| Laboratory | `http://laboratory:8000` | `http://localhost:8001` |
| LLM | `http://llm:8000` | `http://localhost:8002` |
| PostgreSQL | `postgres:5432` | `localhost:5432` |

## Запуск

Из-за названия папки с кириллицей и пробелом Docker Compose нужно запускать с явным именем проекта:

```bash
docker compose -p hospital up --build
```

Остановить сервисы:

```bash
docker compose -p hospital down
```

Запустить только системные тесты через Docker:

```bash
docker compose -p hospital up --build --abort-on-container-exit test
```

## База данных

Схема описана в [schema.sql](schema.sql). Основные таблицы:

- `patients` - пациенты;
- `employees` - сотрудники и врачи;
- `appointments` - записи пациента на прием;
- `cards` - записи электронной медицинской карты;
- `laboratory_investigations` - лабораторные назначения;
- `samples` - лабораторные образцы;
- `analyzers`, `workstations`, `analyzer_results` - выполнение исследования;
- `prescriptions` - электронные рецепты;
- `warehouse_stock`, `dispense_events`, `scanner_events` - склад и выдача лекарств;
- `medical_equipment`, `monitoring_metrics`, `self_diagnostics` - оборудование и мониторинг;
- `llm_reports` - отчеты LLM по результатам анализов.
